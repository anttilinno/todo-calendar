### Key Integration Points

- **Config → Holidays → Calendar:** TOML config loads country → provider initializes with registry → calendar queries holidays per month → grid renders red dates
- **Calendar → Todo Month Sync:** Month navigation triggers `SetViewMonth()` on todolist at 4 locations (init, tab switch, resize, navigation)
- **CRUD → Persistence:** All store mutations call `Save()` with atomic write (CreateTemp + Sync + Rename)
- **Input Mode Isolation:** App-level `isInputting` check suppresses quit/tab during text input

## Tech Debt

**1 item across 1 phase:**

### Phase 3: Todo Management
- **Store.Save() errors ignored in CRUD methods** — `Add()`, `Toggle()`, and `Delete()` call `Save()` but don't propagate errors. If disk write fails, the user gets no feedback and changes are lost on restart. In-memory state remains correct during the session. Severity: minor (disk failures are rare in typical usage, atomic write prevents corruption).
